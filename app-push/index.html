Section 09

Notes / Validation

Data model, join behaviour and excluded rows.

  • Auto / Manual classification: derived directly from the Source / medium column. automation / app-push → Automation Push; manual / app-push → Manual Push.
  • Push name: shown from Manual term (falls back to Session manual term when Manual term is empty / (not set)).
  • Purchasers: each row with a real Transaction ID = 1 purchaser. Rows with (not set) Transaction ID contribute Sessions only.
  • CVR = Total Purchasers / Total Sessions (Total users column was removed from the source table).
  • Current Push / Past Push / Future / Invalid: derived from manual-term pushMMDD patterns relative to the current period.
  • Product join: main table joins to orders_with_date_jan_apr_2026.csv on Transaction ID. Multiple Product titles for the same transaction are concatenated as A | B | C. Missing or unmatched transaction IDs render as Unknown.
  • Per-product attribution: Purchasers per product = count of (Transaction × Product title) rows in table B (one row = one product unit). Revenue / Sessions are not split across multi-product transactions; each product receives the row's full value, so per-product totals can exceed grand totals by design.
